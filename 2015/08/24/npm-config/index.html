npm全局安装失效及变量设置失败

  1. 1. 问题
  2. 2. 可能解决方案

问题

win环境下，nodejs安装使用官网msi文件，npm默认非最新版本；使用npm install -g全局安装时会失效，仍为安装在当前目录下。

可能解决方案

查资料，几种推荐的配置环境方法为：

  1. npm config set prefix "C:\Program Files\nodejs" (node安装目录)
  2. npm config set cache "C:\Program Files\nodejs\cache" (先建好cache目录)
  3. 设置系统环境变量node_path为node_modules目录
    但是使用npm config设置变量时，出现overridden默认值的情况，导致设置不成功，不知道是否为用户权限等问题。
    并因此导致全局安装失效，；因为变量prefix未被设置成功，总是默认为当前目录。
